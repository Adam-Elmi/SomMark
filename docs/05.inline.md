# Inline Syntax

Inline elements are used for shorter, phrase content within your document, such as headers, links, bold text, or custom spans.

## Syntax

```sommark
(Content)->(Identifier)
```

- **Content**: The text or value to be transformed.
- **Identifier**: The target element or function to apply (e.g., `h1`, `bold`, `link`).

## Arguments

Arguments are passed within the second parenthesis, separated by commas or colons depending on the identifier's requirements.

```sommark
(Content)->(Identifier: Arg1, Arg2)
```
---


### ⚠️ Inline Examples

> [!WARNING]
> Most Inline examples do **not** use default mappers.
> They reflect **custom mappers** to demonstrate how value transformations are applied.
> Without a custom mapper registered, behavior may differ from what is illustrated.
> Confirm whether an example depends on a custom mapper before comparing results.


# SomMark – Inline Statement Examples

An Inline statement transforms a **value** using an Identifier.

Inline statements:

* Must be inside a Block
* Cannot exist at the top level
* Cannot be empty
* Do not support key-value arguments

---

## 1. Minimal Inline

```
[Root]
(Hello)->(Bold)
[end]
```

* `Hello` → value
* `Bold` → Identifier

---

## 2. Inline With Arguments

```
[Root]
(Hello)->(Color: red, large)
[end]
```

* `Hello` → value
* `Color` → Identifier
* `red`, `large` → positional arguments

Inline arguments are comma-separated.

---

## 3. Colon Inside Inline Arguments

Inline statements **do not support key-value pairs**.

Colon `:` inside Inline arguments is treated as plain text.

Example:

```
[Root]
(Value)->(Print: x:y:z)
[end]
```

Valid.

Here, `x:y:z` is just a normal argument string.

No escaping required.

---

## 4. Nested Inline Inside Block

Inline must be inside a Block:

```
[Article]
This is (important)->(Bold)
[end]
```

Valid.

---

## 5. Multiple Inline Statements

```
[Message]
(Hello)->(Upper), (world)->(Lower)
[end]
```

Multiple Inline statements can appear in the same Block body.

---

## 6. Inline Value Cannot Be Empty

❌ Invalid:

```
[Root]
()->(Bold)
[end]
```

Reason:
Inline value cannot be empty.

---

## 7. Identifier Rules

Identifiers:

* Letters and numbers only
* No spaces
* No special characters

Valid:

```
(Text)->(Transform123)
```

Invalid:

```
(Text)->(My-Transform)
```

---

## 8. Escaping Inside Inline

To print literal parentheses:

```
[Root]
\((text)\)
[end]
```

To print literal arrow:

```
[Root]
\(value\)->\(Identifier\)
[end]
```

Escape character only affects the next character.

---

## Inline Structure

```
(Value)->(Identifier: optional arguments)
```

Rules:

* Must be inside a Block
* Value cannot be empty
* Arguments optional
* Arguments are positional only
* Colon inside arguments is plain text
* No key-value support
