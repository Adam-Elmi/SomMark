# Built-in Plugins Reference

SomMark comes with several built-in plugins that handle core features. Some are active by default to provide a standard experience, while others are optional.

## 1. Active by Default

These plugins are enabled automatically. You do not need to list them in your configuration.

### `module-system`
Handles modularity by allowing you to import other files and export variables.
- **Hooks**: `preprocessor`
- **Options**: None
- **Documentation**: See [Module System](./19.plugin-system.md) sections in the Plugin System guide for technical details.

### `comment-remover`
Automatically strips all `# comments` from the final output.
- **Hooks**: `beforeLex`
- **Options**: None
- **Note**: To disable this and preserve comments, use `excludePlugins: ["comment-remover"]`.

### `rules-validation`
Validates AST nodes against the `rules` defined in your mapper (e.g., required arguments, value patterns).
- **Hooks**: `on-ast`
- **Behavior**: Throws a descriptive error if validation fails.

---

## 2. Inactive by Default

These plugins must be manually enabled in your `transpile` call or configuration file.

### `raw-content`
Prevents SomMark from parsing the body of specific blocks. Useful for including code snippets, MDX components, or raw HTML.

- **Options**:
  - `targetBlocks` (Array): List of block identifiers to treat as raw. Default: `["mdx", "raw"]`.

**Usage:**
```javascript
const html = await transpile({
  src: "[Code]const x = [1,2,3];[end]",
  plugins: [
    { name: "raw-content", options: { targetBlocks: ["Code"] } }
  ]
});
```

### `sommark-format`
A specialized plugin that takes a SomMark AST and formats it back into a standardized, clean `.smark` string.

- **Options**:
  - `indentString` (String): The character(s) used for indentation. Default: `\t`.

**Usage:**
```javascript
const sm = new SomMark({ plugins: ["sommark-format"] });
await sm.parse(src);
const formatted = sm.plugins.find(p => p.name === "sommark-format").formattedSource;
```

---

## 3. Managing Default Plugins

If you want to disable a plugin that is active by default, use the `excludePlugins` option.

```javascript
const html = await transpile({
  src: mySource,
  excludePlugins: ["comment-remover"]
});
```
