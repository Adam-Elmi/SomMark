# SomMark Syntax Guide

SomMark uses **three main concepts**:

1. **Blocks** – containers for content (can nest other blocks).
2. **Inline Statements** – small content pieces like links, bold text, or icons.
3. **At-Blocks** – raw content that is not processed (e.g., code, math).

---

## 1. Blocks

**Blocks** group content together. They can have text, other blocks, or both.

### Basic Block

```ini
[Block]
This is a simple block.
[end]
```

* `Block` → Identifier
* `[end]` → closes the block

### Block with Arguments

Blocks can take **positional values** or **key-value pairs**.

```ini
[Section = id:about-us, class:dark]
(Welcome)->(h1)
This section has an ID and a class.
[end]
```

* `id:about-us` → key-value
* `class:dark` → key-value
* Positional arguments don’t need keys

> **Tip:** If a value needs a colon (`:`) in a key-value pair, escape it with `\:`
> Example: `[Block = ratio:4\:3][end]`

---

## 2. Inline Statements

**Inline Statements** transform small pieces of text inside a block.

### Basic Inline

```ini
This is (bold text)->(bold).
```

* `(bold text)` → value
* `(bold)` → identifier

### Inline with Arguments

Inline statements can take multiple values separated by commas.

```ini
Click (here)->(link: https://google.com, Go to Google).
```

> **Note:** Inline arguments **do not use key-value pairs**, so colons `:` inside values do **not** need escaping.

---

## 3. At-Blocks

**At-Blocks** are for raw content like code or math. Everything inside is **plain text**.

### Basic At-Block

```ini
@_Code_@: js;
console.log("Hello World");
@_end_@
```

### At-Block with Arguments

Arguments must **end with a semicolon `;`**.

```ini
@_Code_@: javascript;
function hello() {
  return "world";
}
@_end_@
```

---

## 4. Escape Characters

SomMark uses **`\`** to escape reserved characters in content or arguments.

### Examples

* Escaping brackets, parentheses, or at-signs:

```sommark
[Block]
Literal bracket: \[ and \]
Literal parenthesis: \( and \)
Literal at-block marker: \@_ and _\@
[end]
```

* Escaping colon in block arguments:

```ini
[Ratios = scaling:4\:3 eq:2\:1][end]
```

* Escaping comma in inline arguments:

```sommark
(Click Here)->(link: https://example.com/?date\,time)
```

* Escaping equal sign if it’s the first character in a value:

```sommark
[Example = sign : \=1+1][end]
```

* Escaping a backslash:

```js
C:\\Windows\\System32
```

---

## 5. Key Rules

1. **Identifiers**: Letters and numbers only (`Block`, `h1`, `MyComponent`).
2. **Colon `:`**: Key-value separator in block arguments. Escape it `\:` if literal.
3. **Semicolon `;`**: Only for ending At-Block arguments.
4. **Body cannot be empty**: Every block, inline, or At-Block must have content.
5. **Flexible spacing**: Blocks can be written in one line or multiple lines.

### Multi-line Block Example

```ini
[
  Header
  class: main
]
Welcome!
[
  end
]
```
