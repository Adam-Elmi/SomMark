# Core API Reference

The Core API lets you use SomMark **programmatically** in Node.js or browser-based JavaScript projects.

---

## Installation

```bash
npm install sommark
```

---

## 1. `transpile()` – Main Function

Converts SomMark source code into HTML, Markdown, MDX, or plain text.

```javascript
import { transpile } from "sommark";

const output = await transpile({
  src: "[Block]Hello[end]",
  format: "html" // 'html' | 'markdown' | 'mdx' | 'text'
});

console.log(output);
```

### Options

| Option            | Type    | Description                                                          |
| ----------------- | ------- | -------------------------------------------------------------------- |
| `src`             | string  | SomMark source code to compile (**required**)                        |
| `format`          | string  | Output format (`html`, `markdown`, `mdx`, `text`). Default: `'html'` |
| `includeDocument` | boolean | Wrap HTML in `<html><body>...</body></html>`. Default: `true`        |
| `mapperFile`      | Mapper  | Custom Mapper instance for custom tags/blocks                        |

## 2. Lexer & Parser (Advanced)

For advanced usage, you can access **tokens** and the **AST**.

### `lex(source)`

Generates tokens from source code. This method is **asynchronous** as it runs preprocessors:

```javascript
import { lex } from "sommark";

const tokens = await lex("[Block](Hello)->(bold)[end]");
console.log(tokens);
```

### `lexSync(source)`

A **synchronous** version of `lex`. This is a pure lexing function that **does not execute preprocessor plugins**.

```javascript
import { lexSync } from "sommark";

const tokens = lexSync("[Block](Hello)->(bold)[end]");
```

### `parse(source)`

Generates an **Abstract Syntax Tree (AST)**. This method is **asynchronous** as it runs preprocessors and AST plugins:

```javascript
import { parse } from "sommark";

const ast = await parse("[Block]Content[end]");
console.log(ast);
```

### `parseSync(source)`

A **synchronous** version of `parse`. This is a pure parsing function that **does not execute any plugins** (preprocessors, after-lex, or AST plugins).

```javascript
import { parseSync } from "sommark";

const ast = parseSync("[Block]Content[end]");
```

> [!IMPORTANT]
> `lexSync` and `parseSync` run **without plugins**. They are faster but will ignore any preprocessors or custom rules. Use the async version if you need plugin support.

## 3. `SomMark` Class – Object-Oriented Approach

You can also use the `SomMark` class for a more structured approach:

```javascript
import SomMark from "sommark";

const compiler = new SomMark({
  src: "[Block]Hello[end]",
  format: "html"
});

const html = await compiler.transpile();
console.log(html);
```

### Async Plugin Pipeline

Version 3 introduces a multi-stage **asynchronous** plugin pipeline. Methods like `lex()`, `parse()`, and `transpile()` now return Promises because they execute plugins at various stages (pre-lexer, post-lexer, on-ast, and post-transpile).

* `src` → SomMark content
* `format` → Output format
* `transpile()` → Converts the content
