## HTML Mapper
This mapper generates standard HTML5 markup.

### Page Structure

Every SomMark HTML document produces a full HTML5 page. Any block used at the top level contributes its content to the `<body>`. The default page shell is:

```html
<!DOCTYPE html>
<html>
<meta charset="UTF-8" />
<meta name="viewport" content="width=device-width, initial-scale=1.0" />
<meta http-equiv="X-UA-Compatible" content="IE=edge" />
<title>SomMark Page</title>

<body>
  <!-- content from top-level blocks -->
</body>
</html>
```

---

### Predefined Elements

#### 1. Block or block — Transparent Container

A pass-through block. Its body content is rendered as-is with no wrapping HTML element.

**Aliases:** `Block`, `block`

```ini
[Block]
Hello World
[end]
```

This will be interpreted to:

```html
<!DOCTYPE html>
<html>
<meta charset="UTF-8" />
<meta name="viewport" content="width=device-width, initial-scale=1.0" />
<meta http-equiv="X-UA-Compatible" content="IE=edge" />
<title>SomMark Page</title>

<body>
Hello World

</body>
</html>
```

---

#### 2. Html or html — Page Configuration

Configures global page properties. Accepts named (key-value) arguments only. Produces no HTML output on its own.

**Aliases:** `Html`, `html`

| Argument    | Description                          | Default                                  |
|-------------|--------------------------------------|------------------------------------------|
| `title`     | Sets the page `<title>` tag          | `SomMark Page`                           |
| `charset`   | Sets the charset meta tag            | `UTF-8`                                  |
| `viewport`  | Sets the viewport meta content       | `width=device-width, initial-scale=1.0`  |
| `iconSrc`   | URL of the tab favicon               | _(none)_                                 |
| `iconType`  | MIME type of the favicon             | `image/x-icon`                           |
| `httpEquiv` | Value for `X-UA-Compatible` meta     | `IE=edge`                                |

```ini
[Html = title:My Page, charset:UTF-8, iconSrc:/favicon.ico]
[end]
```

---

#### 3. Section or section

Wraps content in a `<section>` tag.

**Aliases:** `Section`, `section`

```ini
[Section]
This is a section.
[end]
```

**Output:**
```html
<section>This is a section.</section>
```

---

#### 4. Headings — h1 to h6

Wraps content in the corresponding HTML heading tag.

**Identifiers:** `h1`, `h2`, `h3`, `h4`, `h5`, `h6`

```ini
[h1]Welcome to SomMark[end]
[h2]A Subheading[end]
```

**Output:**
```html
<h1>Welcome to SomMark</h1>
<h2>A Subheading</h2>
```

---

#### 5. bold or b — Bold Text

Wraps content in a `<strong>` tag.

**Aliases:** `bold`, `b`

```ini
[bold]This is bold.[end]
```

**Output:**
```html
<strong>This is bold.</strong>
```

---

#### 6. italic or i — Italic Text

Wraps content in an `<i>` tag.

**Aliases:** `italic`, `i`

```ini
[italic]This is italic.[end]
```

**Output:**
```html
<i>This is italic.</i>
```

---

#### 7. emphasis or e — Bold + Italic

Wraps content in a `<span>` styled with both bold and italic.

**Aliases:** `emphasis`, `e`

```ini
[emphasis]Important![end]
```

**Output:**
```html
<span style="font-weight:bold; font-style: italic;">Important!</span>
```

---

#### 8. color — Colored Text

Wraps content in a `<span>` with a CSS `color` property.

**Arguments:** `arg1` — the CSS color value (name, hex, rgb, etc.)

```ini
[color = red]This is red text.[end]
```

**Output:**
```html
<span style="color:red">This is red text.</span>
```

---

#### 9. link — Hyperlink

Creates an `<a>` anchor element.

**Arguments:**
- `arg1` (required) — The URL (`href`)
- `arg2` (optional) — The `title` attribute

```ini
[link = https://example.com, Visit Example]Click here[end]
```

**Output:**
```html
<a href="https://example.com" title="Visit Example">Click here</a>
```

---

#### 10. image or Image — Image Element

Renders a self-closing `<img />` tag. Does not have a body.

**Aliases:** `image`, `Image`

**Arguments (named/key-value):**

| Argument | Required | Description      |
|----------|----------|------------------|
| `src`    | ✅ Yes   | Image source URL |
| `alt`    | No       | Alternative text |
| `width`  | No       | Image width      |
| `height` | No       | Image height     |

```ini
[image = src:photo.jpg, alt:A photo, width:800, height:600]
[end]
```

**Output:**
```html
<img src="photo.jpg" alt="A photo" width="800" height="600" />
```

> `src` is required. An error will be thrown if it is missing.

---

#### 11. code or Code — Code Block

Renders a syntax-highlighted `<pre><code>` block. Content is treated as raw text (no SomMark parsing inside).

**Aliases:** `code`, `Code`

**Arguments:** `arg1` (optional) — Language for syntax highlighting (e.g. `javascript`, `python`). Defaults to `text`.

```ini
[code = javascript]
const x = 42;
console.log(x);
[end]
```

**Output:**
```html
<pre><code class="hljs language-javascript">... highlighted code ...</code></pre>
```

> The default syntax highlight theme is `atom-one-dark`.

---

#### 12. list or List — Unordered List

Converts indented plain-text lines into a nested `<ul>` list. Content is treated as raw text. Indentation (2 spaces per level) determines nesting depth.

**Aliases:** `list`, `List`

```ini
[list]
Fruits
  Apples
  Bananas
Vegetables
  Carrots
[end]
```

**Output:**
```html
<ul>
  <li>Fruits<ul><li>Apples</li><li>Bananas</li></ul></li>
  <li>Vegetables<ul><li>Carrots</li></ul></li>
</ul>
```

---

#### 13. table or Table — Data Table

Parses CSV-like content into an HTML `<table>`. Headers are defined as positional arguments; each body line is a row of comma-separated values. Content is treated as raw text.

**Aliases:** `table`, `Table`

**Arguments:** Column headers (positional)

```ini
[table = Name, Age, City]
Alice, 30, New York
Bob, 25, London
[end]
```

**Output:**
```html
<table class="sommark-table">
  <thead><tr><th>Name</th><th>Age</th><th>City</th></tr></thead>
  <tbody>
    <tr><td>Alice</td><td>30</td><td>New York</td></tr>
    <tr><td>Bob</td><td>25</td><td>London</td></tr>
  </tbody>
</table>
```

> Default table styles (`.sommark-table`) are automatically injected into the page when a table is used.

---

#### 14. hr — Horizontal Rule

Renders a self-closing `<hr />` tag. No body, no arguments.

**Identifier:** `hr`

```ini
[hr][end]
```

**Output:**
```html
<hr />
```

---

#### 15. todo — Todo Checkbox

Renders a disabled checkbox with an optional label. The body controls the checked state.

**Identifier:** `todo`

**Arguments:** `arg1` (optional) — Label text next to the checkbox.

**Body values:**
- `x` or `done` (case-insensitive) → checked
- anything else → unchecked

```ini
[todo = Buy groceries]x[end]
[todo = Write docs]pending[end]
```

**Output:**
```html
<div><input type="checkbox" disabled checked />Buy groceries</div>
<div><input type="checkbox" disabled />Write docs</div>
```
