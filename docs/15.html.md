## HTML Mapper
This mapper generates standard HTML5 markup.

### Standard HTML Support
SomMark's HTML mapper fully supports all standard HTML tags. You can use any HTML element as a block.

Example:
```ini
[main]
  [article]
    [header]
      [h1]Hello World[end]
    [end]
    [section]
      This is a section.
    [end]
  [end]
[end]
```

### HTML Attributes
Standard HTML attributes are supported in blocks and inline statements. Attributes are automatically converted to kebab-case where appropriate (e.g., `backgroundColor` becomes `background-color`).

Example:
```ini
[div = class: container, id: main-doc, backgroundColor: #f0f0f0]
  [a = href: https\://example.com, target: _blank]Click here[end]
[end]
```

### Self-Closing Tags
Void elements (like `img`, `br`, `hr`, `input`) are automatically handled. In SomMark block syntax, they require an `[end]` tag.

Example:
```ini
[img = src: logo.png, alt: Logo, width: 200][end]
[br][end]
[hr][end]
```
### CSS & Styling

SomMark provides flexible ways to style your HTML elements.

#### 1. Inline Style Attribute
You can use the standard `style` attribute. If your style string contains commas or colons (common in CSS), use double quotes (handled by the `quote-escaper` plugin).

```ini
[div = style: "color: red; padding: 10px;"]
  Styled content
[end]
```

#### 2. Automatic CSS Property Mapping
For convenience, any attribute that is not a standard HTML property is automatically treated as a CSS property and added to the inline `style` attribute. SomMark supports both camelCase (automatically converted to kebab-case) and direct kebab-case.

```ini
[div = background-color: blue, color: white, border-radius: 5px]
  Direct kebab-case
[end]

[div = backgroundColor: red, paddingLeft: 20px]
  CamelCase auto-conversion
[end]
```

This will produce:
```html
<div style="background-color: blue; color: white; border: 1px solid black;">
  Automatic Styling
</div>
```

### Page Structure

Every SomMark HTML document produces a full HTML5 page. Any block used at the top level contributes its content to the `<body>`. The default page shell is:

```html
<!DOCTYPE html>
<html>
<meta charset="UTF-8" />
<meta name="viewport" content="width=device-width, initial-scale=1.0" />
<meta http-equiv="X-UA-Compatible" content="IE=edge" />
<title>SomMark Page</title>

<body>
  <!-- content from top-level blocks -->
</body>
</html>
```

### Predefined Elements

#### 1. Block or block ‚Äî Transparent Container

A pass-through block. Its body content is rendered as-is with no wrapping HTML element.

**Aliases:** `Block`, `block`

```ini
[Block]
Hello World
[end]
```

This will be interpreted to:

```html
<!DOCTYPE html>
<html>
<meta charset="UTF-8" />
<meta name="viewport" content="width=device-width, initial-scale=1.0" />
<meta http-equiv="X-UA-Compatible" content="IE=edge" />
<title>SomMark Page</title>

<body>
Hello World

</body>
</html>
```

---

#### 2. Html or html ‚Äî Page Configuration

Configures global page properties and CSS variables. Accepts named (key-value) arguments only. Produces no HTML output on its own.

**Aliases:** `Html`, `html`

| Argument    | Description                          | Default                                  |
|-------------|--------------------------------------|------------------------------------------|
| `title`     | Sets the page `<title>` tag          | `SomMark Page`                           |
| `charset`   | Sets the charset meta tag            | `UTF-8`                                  |
| `viewport`  | Sets the viewport meta content       | `width=device-width, initial-scale=1.0`  |
| `iconSrc`   | URL of the tab favicon               | _(none)_                                 |
| `iconType`  | MIME type of the favicon             | `image/x-icon`                           |
| `httpEquiv` | Value for `X-UA-Compatible` meta     | `IE=edge`                                |
| `--*`       | Keys starting with `--` are set as global CSS variables. | _(none)_ |

```ini
[Html = title:My Page, --primary-color:red, iconSrc:/favicon.ico]
[end]
```

#### 5. Headings ‚Äî h1 to h6

Wraps content in the corresponding HTML heading tag.

**Identifiers:** `h1`, `h2`, `h3`, `h4`, `h5`, `h6`

```ini

[h2]A Subheading[end]
```

**Output:**
```html
<h1>Welcome to SomMark</h1>
<h2>A Subheading</h2>
```

#### 9. emphasis ‚Äî Bold + Italic

Wraps content in a `<span>` styled with both bold and italic.



#### 10. color ‚Äî Colored Text

Wraps content in a `<span>` with a CSS `color` property.

**Arguments:** `arg1` ‚Äî the CSS color value (name, hex, rgb, etc.)

```ini
[color = red]This is red text.[end]
```
or
```ini
[Block]
(This is red text.)->(color: red)
[end]
```
**Output:**
```html
<span style="color:red">This is red text.</span>
```

---


#### 11. Code ‚Äî Syntax Highlighted Block

Renders a syntax-highlighted `<pre><code>` block. Content is treated as raw text (no SomMark parsing inside).

**Note:** `Code` (Capitalized) is a special SomMark identifier.

**Arguments:** `arg1` (optional) ‚Äî Language for syntax highlighting (e.g. `javascript`, `python`). Defaults to `text`.

```js
@_Code_@: javascript;
const x = 42;
console.log(x);
@_end_@
```

**Output:**
```html
<pre><code class="hljs language-javascript">... highlighted code ...</code></pre>
```

> The default syntax highlight theme is `atom-one-dark` if method highlightCode is assigned to a function.

---

#### 12. list ‚Äî Unordered List
`list` is special SomMark identifier.
Converts indented plain-text lines into a nested `<ul>` list. Content is treated as raw text. Indentation (2 spaces per level) determines nesting depth.


```lua
@_list_@
Fruits
  Apples
  Bananas
Vegetables
  Carrots
@_end_@
```

**Output:**
```html
<ul>
  <li>Fruits<ul><li>Apples</li><li>Bananas</li></ul></li>
  <li>Vegetables<ul><li>Carrots</li></ul></li>
</ul>
```

---

#### 13. Table ‚Äî Data Table
`Table` is special SomMark identifier.
Parses CSV-like content into an HTML `<table>`. Headers are defined as positional arguments; each body line is a row of comma-separated values. Content is treated as raw text.

**Arguments:** Column headers (positional)

```lua
@_Table_@: Name, Age, City;
Alice, 30, New York
Bob, 25, London
@_end_@
```

**Output:**
```html
<table class="sommark-table">
  <thead><tr><th>Name</th><th>Age</th><th>City</th></tr></thead>
  <tbody>
    <tr><td>Alice</td><td>30</td><td>New York</td></tr>
    <tr><td>Bob</td><td>25</td><td>London</td></tr>
  </tbody>
</table>
```

> Default table styles (`.sommark-table`) are automatically injected into the page when a table is used.

---

#### 14. hr ‚Äî Horizontal Rule

Renders a self-closing `<hr />` tag. No body, no arguments.

**Identifier:** `hr`

```ini
[hr][end]
```

**Output:**
```html
<hr />
```

Injects raw CSS into a `<style>` tag within the document head.

```
@_end_@
```

---

#### 16. script ‚Äî JavaScript Block

Injects raw JavaScript into a `<script>` tag.

**Identifier:** `script`

```ini
@_script_@;

@_end_@
```

---

#### 17. todo ‚Äî Todo Checkbox

Renders a disabled checkbox with an optional label. Supports both block and inline usage.

**Identifier:** `todo`

**Block Usage:**
The body determines the state (`x` or `done` for checked).

```ini
[todo = Buy groceries]done[end]
```

**Inline Usage:**
The identifier determines the state.

```ini
[Block]
(done)->(todo: Buy groceries)
(-)->(todo: Write docs)
[end]
```

**Output:**
```html
<div><input type="checkbox" disabled checked /> Buy groceries</div>
<div><input type="checkbox" disabled /> Write docs</div>
```
