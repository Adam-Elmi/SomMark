# JSON Mapper

The JSON mapper allows you to generate structured, valid JSON output directly from SomMark source files.

Unlike text-based mappers (HTML, Markdown), the JSON mapper processes the **AST (Abstract Syntax Tree)** directly. It doesn't map elements to text snippets; instead, it uses a fixed set of reserved identifiers to build a JSON object or array.

---

## Core Concepts

- **AST-Driven**: The mapper traverses the AST nodes to build the JSON structure.
- **Reserved Identifiers**: Only specific identifiers (`Json`, `Object`, `Array`, `string`, `number`, `bool`, `null`, `array`, `none`) are processed. Other elements are ignored.
- **Auto-Typing**: For inline arrays and the `none` type, SomMark automatically detects and converts values to their native JSON types (number, boolean, null).

> [!IMPORTANT]
> Because it's AST-driven, arbitrary text outside of these reserved blocks/inlines is ignored in the final JSON output.

---

## Root Block: Json

Every JSON document must resolve to a single root. The `Json` block defines whether your output is a root **Object** or a root **Array**.

**Identifier**: `Json`
**Arguments**: `object` (default) or `array`

```ini
[Json = object]
  ...
[end]
```

---

## Structural Blocks

Blocks are used to define nested objects and arrays.

### 1. Object

Defines a JSON object `{}`. 

- **Inside an Object**: Requires a **key** argument (the first positional argument).
- **Inside an Array / Root**: No key argument is needed.

**Identifier**: `Object`

```ini
[Json = object]
  [Object = metadata]
    (version)->(string: 1.0)
  [end]
[end]
```

> [!WARNING]
> Using `[Object]` inside another `Object` without providing a key argument will result in a transpilation error.

---

### 2. Array

Defines a JSON array `[]`.

- **Inside an Object**: Requires a **key** argument.
- **Inside an Array / Root**: No key argument needed.

**Identifier**: `Array`

```ini
[Json = object]
  [Array = tags]
    (-)->(none: sommark, v3, json)
  [end]
[end]
```

---

## Inline Types (Values)

Inline statements define the primitive values and keys within your JSON structure.

**Syntax**: `(key)->(type: value)`

| Type | Description | Example |
| :--- | :--- | :--- |
| `string` | A quoted string | `(name)->(string: SomMark)` |
| `number` | A numeric value | `(count)->(number: 42)` |
| `bool` | `true` or `false` | `(valid)->(bool: true)` |
| `null` | A JSON `null` | `(data)->(null)` |
| `array` | A compact inline array | `(ports)->(array: 80, 443)` |
| `none` | Raw value (Array only) | `(-)->(none: value)` |

### The `none` Type

The `none` identifier is specifically used inside **Arrays** where keys are not required. It can output single values or multiple values (which become a nested array).

```ini
[Json = array]
  # "hello"
  (-)->(none: hello)
  # [1, 2, 3]
  (-)->(none: 1, 2, 3)
[end]
```

> [!CAUTION]
> The `none` type is **not allowed** directly inside an `Object`.

---

## Auto-Typing Rules

For the `array` and `none` types, SomMark attempts to cast arguments to their native types:

| Input Value | JSON Output Type |
| :--- | :--- |
| `null` | `null` |
| `true` / `false` | `boolean` |
| `42`, `3.14` | `number` |
| `"hello"` | `string` |

---

## Nesting Rules

| Context | Allowed Children | Rule |
| :--- | :--- | :--- |
| `Json` | `Object`, `Array`, Inlines | Root level container |
| `Object` | `Object`, `Array`, Inlines | Every child **must** have a key |
| `Array` | `Object`, `Array`, `none` | Children **cannot** have keys |

## Full Example: Configuration File

```ini
[Json = object]
  [Object = compilerOptions]
    (target)->(string: es2022)
    (strict)->(bool: true)
    (baseUrl)->(null)
    
    [Array = paths]
      [Object]
        (@/*)->(array: ./src/*)
      [end]
    [end]
  [end]

  (include)->(array: src/**/*)
  (exclude)->(array: node_modules, dist)
[end]
```

**Output**:
```json
{
  "compilerOptions": {
    "target": "es2022",
    "strict": true,
    "baseUrl": null,
    "paths": [
      {
        "@/*": ["./src/*"]
      }
    ]
  },
  "include": ["src/**/*"],
  "exclude": ["node_modules", "dist"]
}
```

---

## Output Validation

The JSON mapper includes a built-in safety check. Before returning the result, it attempts to `JSON.parse()` the generated string to ensure validity. If the structure is broken (e.g., missing keys in an object), a `JSON Format Error` is thrown with a detailed reason.
