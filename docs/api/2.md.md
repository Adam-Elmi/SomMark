# md

The `md` property exposes an instance of the `MarkdownBuilder` class. Use it to programmatically generate Markdown strings within your custom mappers.

**Syntax:** `mapper.md` (MarkdownBuilder)

---

## Methods

### `heading(text, level)`
Creates a Markdown heading.

-   **text** (string): The heading text.
-   **level** (number | string): The heading level (1-6). Default: 1.

```js
const h1 = mapper.md.heading("Introduction", 1); // "# Introduction\n"
const h3 = mapper.md.heading("Details", 3);      // "### Details\n"
```

### `bold(text, is_underscore)`
Makes text bold.

-   **text** (string): The text to bold.
-   **is_underscore** (boolean): If `true`, uses `__` instead of `**`.

```js
const strong = mapper.md.bold("Important"); // "**Important**"
```

### `italic(text, is_underscore)`
Makes text italic.

-   **text** (string): The text to italicize.
-   **is_underscore** (boolean): If `true`, uses `_` instead of `*`.

```js
const em = mapper.md.italic("Note"); // "*Note*"
```

### `codeBlock(code, language)`
Creates a fenced code block.

-   **code** (string | Array): The code content.
-   **language** (string): Optional language identifier.

```js
const block = mapper.md.codeBlock("console.log('Hi')", "js");

// ```js
// console.log('Hi')
// ```
```

### `url(type, text, url, title)`
Creates a link or image.

-   **type** (string): "image" for `![]()`, empty for `[]()`.
-   **text** (string): Link text or alt text.
-   **url** (string): The URL.
-   **title** (string): Optional tooltip title.

```js
const link = mapper.md.url("", "Home", "/"); // "[Home](/)"
const img = mapper.md.url("image", "Logo", "logo.png"); // "![Logo](logo.png)"
```

### `table(headers, rows)`
Creates a Markdown table.

-   **headers** (Array\<string>): Header columns.
-   **rows** (Array): Data rows.

```js
const table = mapper.md.table(
  ["ID", "Name"],
  ["1, Alice", "2, Bob"]
);
```

### `todo(checked, text)`
Creates a task list item.

-   **checked** (boolean): `true` for `[x]`, `false` for `[ ]`. Default: `false`.
-   **text** (string): The task description.

```js
const task = mapper.md.todo(true, "Buy milk"); // "- [x] Buy milk\n"
```
### Recommended Way to Handle Todo

When creating a custom `todo` mapper, it is recommended to use the inline content for the state and the argument for the text. Why? Because it is more readable and it is easier to understand what is going on.

```js
mapper.register("todo", ({ args, content }) => {
    const checked = content === "x" ? true : false;
  const text = args[0] || "";
  return mapper.md.todo(checked, text);
});

// Example: (x)->(todo: Unchecked Item)
// x = content
// Unchecked Item = args[0]
```

**Usage:**
```ini
(-)->(todo: Unchecked Item)
(x)->(todo: Checked Item)
```


### `horizontal(format)`
Creates a horizontal rule.

-   **format** (string): Style of rule (`*`, `_`, etc.). Default: `*`.

```js
const hr = mapper.md.horizontal(); // "\n***\n"
```

### `escape(text)`
Escapes Markdown special characters.

-   **text** (string): The raw text.

```js
const safe = mapper.md.escape("[No Link]"); // "\[No Link\]"
```
