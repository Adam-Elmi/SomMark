# safeArg

A robust helper for accessing and validating arguments passed to a block. When inside a `register()` callback, you should access it via `this.safeArg`.

**Syntax:** `this.safeArg(args, index?, key?, type?, setType?, fallBack?)`

### Parameters
- `args`: The argument object from the mapper callback.
- `index` (number): Optional positional index (0-based).
- `key` (string): Optional named argument key.
- `type` (string): Optional type to validate (e.g., `"string"`, `"number"`).
- `setType` (function): Optional transformer (e.g., `Number`, `Boolean`).
- `fallBack`: Value to return if the argument is missing or invalid.

---

### Example: Positional Argument
Handles `[YouTube = abc123][end]`.

```js
HTML.register("YouTube", function({ args }) {
  const id = this.safeArg(args, 0, null, "string");
  return `<iframe src=".../${id}"></iframe>`;
});
```

### Example: Named Argument
Handles `[YouTube = id:abc123][end]`.

```js
const id = this.safeArg(args, undefined, "id", "string");
```

### Example: Type Transformation & Fallback
Handles `[Box = width:500][end]`.

```js
HTML.register("Box", function({ args, content }) {
  const width = this.safeArg(args, 0, "width", "number", Number, 100);
  return `<div style="width: ${width}px">${content}</div>`;
});
```

> [!TIP]
> `safeArg` will check the `index` first, and if not found, it will try the `key`. This allows your blocks to support both positional and named arguments simultaneously.
