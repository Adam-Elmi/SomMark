# inherit

The `inherit` method allows a mapper to import all registered outputs from one or more other mappers. This is the primary way to achieve composition and reuse common block definitions.

**Syntax:** `mapper.inherit(...mappers)`

-   **...mappers** (Mapper): One or more Mapper instances to inherit from.

### How it Works
1.  It iterates through all `outputs` of the target mapper.
2.  For each inherited output, it first removes any existing output with the same ID in the current mapper to prevent duplicates.
3.  It then pushes the inherited output into the current mapper's `outputs` list.

### Example: Composition
Inherit from a standard mapper and then extend it with custom blocks.

```js
import { Mapper, HTML } from "sommark";

const myMapper = new Mapper();

// Inherit all standard HTML blocks
myMapper.inherit(HTML);

// Overwrite a specific block or add a new one
myMapper.register("bold", function({ content }) {
  return `<b>${content}</b>`;
});
```

> [!NOTE]
> Inherited blocks that share IDs with existing ones will **overwrite** the previously registered versions.
