# extraProps

The `extraProps` property is a `Set` that stores globally recognized argument keys. This is primarily used by plugins to declare arguments that should be allowed across all blocks, bypassing strict local validation rules.

**Type:** `Set<string>`

### How it Works
When a block is being validated (e.g., via the `includes` rule in `RulesValidationPlugin`):
1.  The validator checks if an argument key is in the block's local `includes` list.
2.  If not found, it checks if the key exists in the mapper's `extraProps` set.
3.  If found in either, the argument is considered valid.

### Example: Global Plugin Property
A plugin that adds a `class` attribute support to all blocks.

```js
import { HTML } from "sommark";

// Register 'class' as a globally recognized argument
HTML.extraProps.add("class");

// Now even blocks with restricted 'includes' will accept 'class'
HTML.register("Box", function({ content, args }) {
  const cls = this.safeArg(args, undefined, "class", "string", null, "");
  return `<div class="${cls}">${content}</div>`;
}, {
  rules: { includes: ["id"] } // 'class' is allowed even if not listed here
});
```

> [!IMPORTANT]
> Arguments in `extraProps` are only "allowed" by the validation system; your block's renderer must still explicitly handle them (e.g., via `this.safeArg`).
