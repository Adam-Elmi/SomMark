# getUnknownTag

The `getUnknownTag` method is a hook that allows mappers to define a fallback rendering strategy for tags that have not been explicitly registered via `register()`.

**Syntax:** `mapper.getUnknownTag(node)`

-   **node** (Object): The AST node representing the unknown tag.
-   **Returns**: A string (HTML/Markdown) or `null`.

### How it Works
When the transpiler encounters a tag that isn't in the `outputs` registry, it calls this method.
-   If it returns a **string**, that string is used as the output for that node.
-   If it returns **`null`**, the transpiler will instead throw an "unrecognized tag" error.

### Example: Auto-Wrapping
Create a mapper that automatically wraps unknown tags in a generic `<span>`.

```js
import { Mapper } from "sommark";

class MyMapper extends Mapper {
  getUnknownTag(node) {
    // Wrap unknown tags in a span with the tag name as a class
    return this.tag("span")
      .attributes({ class: `unknown-${node.id.toLowerCase()}` })
      .body(node.content);
  }
}
```

> [!TIP]
> This is particularly useful for mappers like MDX, where you might want to render custom components as-is even if they aren't registered.
