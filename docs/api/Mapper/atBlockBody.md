# atBlockBody()

Filters and processes the raw, literal content of At-Blocks.
---

**Syntax:**
```js
mapper.atBlockBody(text, [options]);
```

**Usage:**
```js
import { Mapper } from "sommark";

const mapper = Mapper.define({
  atBlockBody(text, options) {
    if (options?.escape !== false) {
      return this.escapeHTML(text);
    }
    return text;
  }
});
```

---

### Example: Escaping Inside a Custom HTML Code Block

When an At-Block like `code` is transpiled, the raw body content is filtered through `atBlockBody`.

```javascript
import { transpile, HTML } from "sommark";

const mapper = HTML.clone();

// Escape HTML markers unless specifically disabled in the tag's options
mapper.atBlockBody = function (text, options) {
  return options?.escape !== false ? this.escapeHTML(text) : text;
};

// Register a code tag using the fluent tag builder API
mapper.register("code", function ({ content }) {
  return this.tag("pre").body(this.tag("code").body(content));
}, { type: "AtBlock", escape: true });

const smarkSource = `
@_code_@;
<div class="card">
  <h1>Title</h1>
</div>
@_end_@
`;

const output = await transpile({
  src: smarkSource,
  format: "html",
  mapperFile: mapper,
});

console.log(output);
```

**Output:**
```html
<pre><code>&lt;div class=&quot;card&quot;&gt;
  &lt;h1&gt;Title&lt;/h1&gt;
&lt;/div&gt;</code></pre>
```

### Understanding the `options` Argument

The `options` argument is **completely custom**. You can define any static properties (booleans, strings, functions, objects) during tag registration, and access them dynamically during filtering.

#### Rich Examples of Custom Options

##### 1. Configurable Line Numbers (Filtering content dynamically)
You can use a custom boolean option to dynamically format the raw text:
```javascript
// 1. Process block text dynamically based on tag options
mapper.atBlockBody = function (text, options) {
  if (options?.lineNumbers) {
    return text.split("\n").map((line, idx) => `${idx + 1}: ${line}`).join("\n");
  }
  return text;
};

// 2. Register tag with static option
mapper.register("code", function ({ content }) {
  return this.tag("pre").body(content);
}, { type: "AtBlock", lineNumbers: true });
```
*Input:*
```r
@_code_@;
const x = 1;
const y = 2;
@_end_@
```
*Output:*
```html
<pre>1: const x = 1;
2: const y = 2;</pre>
```

##### 2. Custom Comment Injection (Prepending metadata dynamically)
You can read custom options to prepend a header notice or comment to the text:
```javascript
// 1. Process block text dynamically based on tag options
mapper.atBlockBody = function (text, options) {
  if (options?.commentHeader) {
    return `<!-- ${options.commentHeader} -->\n` + text;
  }
  return text;
};

// 2. Register tag with static comment header option
mapper.register("raw", function ({ content }) {
  return this.tag("div").body(content);
}, { type: "AtBlock", commentHeader: "Generated by SomMark" });
```
*Input:*
```r
@_raw_@;
Hello World
@_end_@
```
*Output:*
```html
<div><!-- Generated by SomMark -->
Hello World</div>
```

### Default Engine Behaviors
* **Implicit Trimming:** The transpiler automatically absolute-dedents the At-Block body based on its starting indentation level and trims surrounding leading/trailing whitespace before sending it to `atBlockBody()`.
* **Base Mapper Fallback:** The base `Mapper` class returns the text exactly as-is: `atBlockBody(text) { return text; }`.
