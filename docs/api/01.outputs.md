# outputs

A list storing all registered block and inline mappings. Using this array directly is generally discouraged; use methods like `register()`, `inherit()`, and `removeOutput()` for a safer API.

**Syntax:** `mapper.outputs` (Array)

### Item Structure
Each item in the array is an object with the following properties:
- `id` (string | Array): Unique identifier(s) for the tag.
- `render` (function): The bound renderer function.
- `options` (object): Block settings (e.g., `{ escape: true }`).

```js
console.log(mapper.outputs[0].id); // "bold"
```

### Composing Mappers
The recommended way to combine outputs from different mappers is using the `inherit()` method.

```js
import { Mapper, HTML } from 'sommark';

const myMapper = new Mapper();

// Safe way to copy all outputs from HTML mapper
myMapper.inherit(HTML);
```

### Manual Registration from Other Mappers
You can still extract specific mappings and register them elsewhere.

```js
import { Mapper, HTML } from 'sommark';

const myMapper = new Mapper();
const codeOutput = HTML.get("Code");

if (codeOutput) {
    myMapper.register(codeOutput.id, codeOutput.render, codeOutput.options);
}
```

> [!TIP]
> Use `mapper.get(id)` to safely retrieve an output object by its ID.
