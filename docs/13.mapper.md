# Mapper API Reference (Simplified)

The **Mapper** class is the core of SomMark’s extensibility. It lets you create **custom blocks, inline statements, or At-Blocks** and define how they render into HTML, Markdown, or other formats.

```javascript
import { Mapper } from "sommark";

const myMapper = new Mapper();
```

---

## 1. Register Custom Tags

Use `register()` to define new blocks or inline elements.

**Syntax:**

```javascript
mapper.register("Identifier", RendererFunction, options?)
```

- **Identifier** → Name of the block or inline
- **RendererFunction** → Function that returns output (HTML/Markdown)
- **options** → Optional rules and settings

### Example: YouTube Block

```javascript
myMapper.register("YouTube", ({ args }) => {
	return myMapper
		.tag("iframe")
		.attributes({ src: `https://youtube.com/embed/${args[0]}` })
		.body("");
});
```

**SomMark usage:**

```ini
[YouTube = dQw4w9WgXcQ][end]
```

### Example: Alert Block

```javascript
myMapper.register("Alert", ({ content }) => {
	return `<div class="alert alert-warning">${content}</div>`;
});
```

```ini
[Alert]Warning! Something went wrong.[end]
```

---

## 2. Mapper Inheritance

You can combine multiple mappers using `inherit()`. This is useful for building upon standard mappers (like HTML) with your own custom tags.

```javascript
import { Mapper, HTML } from "sommark";

const myMapper = new Mapper();
myMapper.inherit(HTML); // Now includes all standard HTML tags

myMapper.register("Custom", () => "<span>Custom</span>");
```

## 3. Generic Tag Handling (Unknown Tags)

Use `getUnknownTag()` to define how SomMark should handle tags that haven't been explicitly registered.

```javascript
myMapper.getUnknownTag = (node) => {
    // node contains { id, args, content, type }
    return `<div class="unknown">${node.id}</div>`;
};
```

### Argument Rules

Control what arguments are allowed:

```javascript
myMapper.register(
	"User",
	({ args }) => {
		return `<div>User: ${args[0]}</div>`;
	},
	{
		rules: {
			args: {
				min: 1, // At least 1 argument
				max: 3, // Maximum 3 arguments
				required: ["id"], // Key "id" is required
				includes: ["id", "role", "age"] // Only these keys allowed
			}
		}
	}
);
```

### Content Rules

Control the block body:

```javascript
myMapper.register(
	"Summary",
	({ content }) => {
		return `<p>${content}</p>`;
	},
	{
		rules: {
			content: { maxLength: 100 } // Max 100 characters
		}
	}
);
```

### Self-Closing Blocks

Ensure a block has **no content**:

```javascript
myMapper.register("Separator", () => `<hr/>`, {
	rules: { is_self_closing: true }
});
```

### Element Type

Ensure element type has to be expected type:

* Type that can be passed: `Block` | `Inline` | `AtBlock`

```javascript
myMapper.register("Summary", ({content}) => {
  return myMapper.tag("p").body(content);
});
```

## 4. Argument Handling (`safeArg`)

SomMark provides a robust helper for retrieving and validating arguments.

```javascript
myMapper.register("Heading", ({ args, content }) => {
    // safeArg(args, index, key, type, setType, fallBack)
    const level = myMapper.safeArg(args, 0, "level", "number", Number, 1);
    const id = myMapper.safeArg(args, undefined, "id", "string", null, null);
    
    return `<h${level} id="${id}">${content}</h${level}>`;
});
```

## 5. Style & Header Management

Mappers can manage global styles and HTML metadata.

```javascript
// Add raw CSS string or object
myMapper.addStyle(".my-class { color: red; }");
myMapper.addStyle({ ".my-class": { "font-weight": "bold" } });

// Load from file (Async)
await myMapper.loadCss("node", "./styles.css");

// Update Page Metadata
myMapper.pageProps.pageTitle = "My New Title";
myMapper.pageProps.charset = "UTF-8";

// Add Raw Header Content
myMapper.setHeader(["<link rel='preconnect' href='...'>"]);
```

## 6. Format Hooks

These methods are designed to be overridden by language-specific mappers to handle comments and final output wrapping.

### Comment Handling (`comment`)
Defines how SomMark comments should be rendered in the target format. Note that the `text` argument includes the leading `#` character by default.

```javascript
myMapper.comment = (text) => {
    // Strip the leading '#' and wrap in HTML comments
    return `<!-- ${text.replace("#", "").trim()} -->`;
};
```

### Output Formatting (`formatOutput`)
Triggered at the end of the transpilation process. It can be used to wrap the output in a full document structure or perform post-processing.

```javascript
myMapper.formatOutput = (output, includeDocument) => {
    if (includeDocument) {
        return `<html><body>${output}</body></html>`;
    }
    return output;
};
```

---

## 7. Helper Methods & Builders

Mappers include built-in utilities for generating content safely.

| Method | Description |
| --- | --- |
| `tag(name)` | Access **TagBuilder** for safe HTML/JSX generation. |
| `md` | Access **MarkdownBuilder** for Markdown generation. |
| `comment(text)` | Define how comments are rendered. |
| `formatOutput(out, doc)` | Final post-processing hook for the output. |
| `code(args, content)` | Pre-built code block renderer with highlighting support. |
| `htmlTable(data, headers)` | Format a table from a string or array. |
| `list(data, as)` | Format a list (`ul`/`ol`) from a string. |
| `clone()` | Create a deep copy of the mapper instance. |
| `clear()` | Wipe all registered mappings. |
| `removeOutput(id)` | Remove a specific mapping by ID. |

---

## 7. Advanced Properties

- `outputs`: List of all registered rendering rules.
- `extraProps`: A `Set` of argument keys recognized by the mapper (used by plugins).
- `highlightCode`: A function hook for providing custom syntax highlighting logic.
