## MDX Mapper
This mapper generates [MDX](https://mdxjs.com/) output — Markdown extended with JSX.

### Standard HTML & JSX Elements
In SomMark v3, the MDX mapper automatically handles **any identifier** as a JSX component or HTML tag. You don't need to register tags unless you want custom logic.

```ini
[Layout = title: Home]
  [Hero]
    # Standard HTML tags are also rendered as JSX
    [h1]Welcome to MDX[end]
    [Button = primary: true]Get Started[end]
  [end]
[end]
```

**Output:**
```jsx
<Layout title="Home">
  <Hero>
    <h1>Welcome to MDX</h1>
    <Button primary={true}>Get Started</Button>
  </Hero>
</Layout>
```

---

### JSX Attribute Handling
SomMark automatically maps attributes to JSX props with smart type detection:

1.  **Strings**: Basic values remain strings.
2.  **Numbers & Booleans**: `[MyTag = count: 42, active: true][end]` becomes `<MyTag count={42} active={true} />`.
3.  **Expressions**: Mathematical expressions are detected and wrapped in curly braces.
4.  **CamelCase Conversion**: Attributes like `backgroundColor` become `style={{ backgroundColor: ... }}` if they match CSS properties.
5.  **className**: The `class` attribute is automatically converted to `className`.
6.  **Events**: Attributes starting with `on` are treated as JSX expressions (e.g., `onClick: handleClick` -> `onClick={handleClick}`).

```ini
[Card = class: shadow-lg, padding: 20, onHover: animate]
  Content
[end]
```

**Output:**
```jsx
<Card className="shadow-lg" padding={20} onHover={animate}>
  Content
</Card>
```

### Standard Markdown Support
The MDX mapper inherits **all elements from the [Markdown mapper](./markdown.md)** (bold, italic, list, etc.). These are rendered as standard Markdown within your MDX file.

```ini
[Block]
  # This is standard Markdown
  (Bold text)->(bold)
  - List item
[end]
```

---

### Raw MDX Content
Use the `mdx` block to inject raw MDX code (like imports or exports) that shouldn't be parsed by SomMark. This feature is powered by the `raw-content-plugin`, which ensures the block content is treated as plain text during lexing.

```js
[mdx]
import { Chart } from './components/Chart';

export const metadata = { date: '2024-03-15' };
[end]
```

---

### Advanced: Custom MDX Components
You can still register custom logic using `MDX.register()` combined with the `tag()` builder.

```js
MDX.register("Alert", ({ args, content }) => {
  return MDX.tag("Alert")
    .props([
      { type: "string", variant: args[0] ?? "info" },
      { type: "other",  dismissible: args[1] ?? "false" }
    ])
    .body(content);
});
```

### Summary Table

| Identifier(s) | MDX Output | Has Body | Args |
|---------------|------------|----------|------|
| `Block`       | _(pass-through)_ | Yes | None |
| `mdx`         | _(raw MDX)_ | Yes (raw) | None |
| `h1` – `h6`   | `<h1>` – `<h6>` | Yes | None |
| `bold`        | `**...**` | Yes | None |
| `italic`      | `*...*` | Yes | None |
| `url`         | `[text](url)` | Yes | Positional |
| `image`       | `![alt](url)` | Yes | Positional |
| `Table`       | `|...|`       | Yes (raw) | Positional |
| `AnyIdentifier`| `<AnyIdentifier />` | Yes | Named |
