## MDX Mapper
This mapper generates [MDX](https://mdxjs.com/) output — Markdown extended with JSX.

The MDX mapper inherits **all elements from the [Markdown mapper](./markdown.md)** with identical syntax and behavior. Refer to the Markdown mapper documentation for the full element reference.

### Custom MDX Components

You can register custom MDX components using `MDX.register()` combined with the `tag()` and `props()` builder methods. This lets you produce JSX component output from SomMark syntax — no raw MDX required.

#### TagBuilder Props

The `tag()` builder exposes a `.props()` method that accepts typed prop entries:

| `type` value | Output format     | Use case                     |
|--------------|-------------------|------------------------------|
| `"string"`   | `key="value"`     | String props                 |
| `"other"`    | `key={value}`     | Expressions, booleans, numbers |

#### Example: Registering a Custom Component

```js
import MDX from "sommark/mappers/languages/mdx.js";

MDX.register("Alert", ({ args, content }) => {
  return MDX.tag("Alert")
    .props([
      { type: "string", variant: args[0] ?? "info" },
      { type: "other",  dismissible: args[1] ?? "false" }
    ])
    .body(content);
});
```

With this registered, you can use it in `.smark` files:

```ini
[Alert = warning, true]
This is a warning message.
[end]
```

**Output:**
```mdx
<Alert variant="warning" dismissible={true}>This is a warning message.</Alert>
```

#### Self-Closing Components

Use `.selfClose()` for components that take no children:

```js
MDX.register("Divider", ({ args }) => {
  return MDX.tag("Divider")
    .props([{ type: "string", style: args[0] ?? "solid" }])
    .selfClose();
});
```

```ini
[Divider = dashed][end]
```

**Output:**
```mdx
<Divider style="dashed" />
```
