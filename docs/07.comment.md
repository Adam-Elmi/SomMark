# SomMark Comments

SomMark supports comments similar to Python, but when transpiled, comments are converted into the **target format’s comment style** if the format supports comments.

---

## 1. Single-Line Comments

Start a line with `#`:

```ini
# This is a comment
[Block]
Hello World!
[end]
```

**Behavior:**

* In **HTML**, transpiled as:

```html
<!-- This is a comment -->
<div>Hello World!</div>
```

* In **Markdown**, transpiled as:

```markdown
<!-- This is a comment -->
Hello World!
```

* In **MDX**, transpiled as:

```mdx
{/* This is a comment */}
Hello World!
```

* In **Plain Text**, comments are usually **removed** or ignored.

---

## 2. Inline Comments

Comments can also appear at the end of a line:

```ini
[Block]
Hello World!  # This is an inline comment
[end]
```

* The inline comment is transpiled in the same way as a top-level comment.

---

## 3. Preservation and Stripping

By default, SomMark **removes** all comments from the final output using the built-in `comment-remover` plugin. This keeps your production files clean and minimal.

### Preserving Comments
If you want to keep comments in your output (converted to the target format's syntax), you must explicitly exclude the remover:

```javascript
import { transpile } from "sommark";

const html = await transpile({
  src: "[Block]# Maintenance Note\nUpdate this section[end]",
  excludePlugins: ["comment-remover"] // Disables the remover
});
```

---

## 4. Syntax Rules

1. **Structural**: Comments can appear anywhere (top-level or inside blocks).
2. **Literal Hash**: A `#` inside **strings or arguments** is treated as literal text.
3. **Escaping**: Use `\#` to print a literal hash at the start of a line or after spaces.

> [!TIP]
> Comment stripping is the standard behavior for v3. If you need preserved comments for documentation purposes, use the `excludePlugins` option.
