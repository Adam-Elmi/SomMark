## Markdown Mapper
This mapper generates standard Markdown output.

### Predefined Elements

#### 1. Block — Transparent Container

A pass-through block. Its body content is rendered as-is with no Markdown wrapping.

**Identifier:** `Block`

```ini
[Block]
Hello World
[end]
```

**Output:**
```
Hello World
```

---

#### 2. Section — Transparent Section

Same as `Block` — content is passed through unchanged. Useful for grouping semantically related content.

**Identifier:** `Section`

```ini
[Section]
Some content here.
[end]
```

**Output:**
```
Some content here.
```

---

#### 3. Heading — Heading with Level Argument

Renders a Markdown heading at a specified level. Takes two positional arguments: level and text.

**Identifier:** `Heading`

**Arguments:**
- `arg1` — Heading level (`1`–`6`)
- `arg2` — Heading text

```ini
[Heading = 2, Introduction]
Some intro content.
[end]
```

**Output:**
```markdown
## Introduction
Some intro content.
```

> Level is clamped between 1 and 6 automatically.

---

#### 4. h1 – h6 — Inline Headings

Shorthand headings where the identifier determines the level and the body is the heading text.

**Identifiers:** `h1`, `h2`, `h3`, `h4`, `h5`, `h6`

```ini
[Block]
(Page Title)->(h1)
(A Subsection)->(h3)
# Or Block with h1 and h3
[h1]Page Title[end]
[h3]A Subsection[end]
[end]
```

**Output:**
```markdown
# Page Title
### A Subsection
```

---

#### 5. bold or b — Bold Text

Wraps content with `**...**`.

**Aliases:** `bold`, `b`

```ini
[Block]
(This is bold)->(bold)
# Or Block with bold
[bold]This is bold[end]
[end]
```

**Output:**
```markdown
**This is bold.**
```

---

#### 6. italic or i — Italic Text

Wraps content with `*...*`.

**Aliases:** `italic`, `i`

```ini
[Block]
(This is italic)->(italic)
# Or Block with italic
[italic]This is italic[end]
[end]
```

**Output:**
```markdown
*This is italic.*
```

---

#### 7. emphasis or e — Bold + Italic

Wraps content with `***...***`.

**Aliases:** `emphasis`, `e`

```ini
[Block]
(Very important!)->(emphasis)
# Or Block with emphasis
[emphasis]Very important![end]
[end]
```

**Output:**
```markdown
***Very important!***
```

---

#### 8. code or Code or codeBlock or CodeBlock — Code Block

Renders a fenced Markdown code block. Content is treated as raw text.

**Aliases:** `code`, `Code`, `codeBlock`, `CodeBlock`

**Arguments:** `arg1` (optional) — Language identifier for syntax highlighting.

```ini
@_code_@: js;
const x = 42;
console.log(x);
@_end_@
```

**Output:**
````markdown
```javascript
const x = 42;
console.log(x);
```
````

---

#### 9. link — Hyperlink

> [!Tip]:
> Use inline statements for links

Renders a Markdown inline link `[text](url "title")`.

**Identifier:** `link`

**Arguments:**
- `arg1` (required) — URL
- `arg2` (optional) — Link title

Body is the link display text.

```ini
[Block]
(Click here)->(link: https://example.com, Visit Example)
[end]
```

**Output:**
```markdown
[Click here](https://example.com "Visit Example")
```

---

#### 10. image — Image

> [!Tip]:
> Use inline statements for images

Renders a Markdown inline image `![alt](url "title")`.

**Identifier:** `image`

**Arguments:**
- `arg1` (required) — Image URL
- `arg2` (optional) — Image title

Body is the alt text.

```ini
[Block]
(Landscape)->(image: photo.jpg, A nice photo)
[end]
```

**Output:**
```markdown
![Landscape](photo.jpg "A nice photo")
```

---

#### 11. horizontal or h — Horizontal Rule

Renders a Markdown horizontal rule. The body controls the format character.

**Aliases:** `horizontal`, `h`

**Body values:**
- `*` → `***` (default)

```ini
[hr][end]
[hr = _][end]
[hr = *][end]
[hr = -][end]
```

**Output:**
```markdown
***
___
***
---
```

---

#### 12. escape or s — Escape Special Characters

Escapes all Markdown special characters in the content: `\ * _ { } [ ] ( ) # + - . ! > |`

**Aliases:** `escape`, `s`

```ini
[escape]Hello *World* _test_[end]
```

**Output:**
```markdown
Hello \*World\* \_test\_
```

---

#### 13. table — Table

Renders a Markdown GFM table. Headers are defined as positional arguments; each body line is a row of comma-separated values. Content is treated as raw text.

**Identifier:** `table`

**Arguments:** Column headers (positional)

```ini
[table = Name, Age, City]
Alice, 30, New York
Bob, 25, London
[end]
```

**Output:**
```markdown
| Name  | Age | City     |
|-------|-----|----------|
| Alice | 30  | New York |
| Bob   | 25  | London   |
```

---

#### 14. list or List — List

Passes content through as-is. Content is treated as raw text, so you write standard Markdown list syntax directly in the body.

**Aliases:** `list`, `List`

```ini
[list]
- Apples
- Bananas
  - Cavendish
- Oranges
[end]
```

**Output:**
```markdown
- Apples
- Bananas
  - Cavendish
- Oranges
```

---

#### 15. todo — Todo Checkbox

Renders a Markdown GFM task list item.

**Identifier:** `todo`

**Arguments:** `arg1` — The label text for the todo item.

**Body values:**
- `x` → checked (`- [x]`)
- anything else → unchecked (`- [ ]`)

```ini
[todo = Buy groceries]x[end]
[todo = Write docs]pending[end]
```

**Output:**
```markdown
- [x] Buy groceries
- [ ] Write docs
```
