## Markdown Mapper
This mapper generates standard Markdown output.

### Standard HTML Support
The Markdown mapper inherits support for all standard HTML tags from the [HTML mapper](./html.md). This allows you to use SomMark syntax to generate raw HTML elements directly within your Markdown output.

Note that Markdown-specific elements (like headings and bold) take precedence over their HTML counterparts.

```ini
[div = background-color: #f0f0f0, padding: 10px]
  This is a div rendered as raw HTML in Markdown.
  [span = color: red]Red text[end]
[end]
```

**Output:**
```html
<div style="background-color: #f0f0f0; padding: 10px;">
  This is a div rendered as raw HTML in Markdown.
  <span style="color: red;">Red text</span>
</div>
```

### Predefined Elements

#### 1. Block — Transparent Container

A pass-through block. Its body content is rendered as-is with no Markdown wrapping.

**Identifier:** `Block`

```ini
[Block]
Hello World
[end]
```

**Output:**
```
Hello World
```

---

#### 2. h1 – h6 — Headings

Block-type headings where the identifier determines the level and the body is the heading text.

**Identifiers:** `h1`, `h2`, `h3`, `h4`, `h5`, `h6`

```ini
[h1]Page Title[end]
[h3]A Subsection[end]
```

**Output:**
```markdown
# Page Title
### A Subsection
```

---

#### 3. bold — Bold Text

Wraps content with `**...**`.

**Identifier:** `bold`

```ini
[bold]This is bold[end]
```

**Output:**
```markdown
**This is bold.**
```

---

#### 4. italic — Italic Text

Wraps content with `*...*`.

**Identifier:** `italic`

```ini
[italic]This is italic[end]
```

**Output:**
```markdown
*This is italic.*
```

---

#### 5. emphasis — Bold + Italic

Wraps content with `***...***`.

**Identifier:** `emphasis`

```ini
[emphasis]Very important![end]
```

**Output:**
```markdown
***Very important!***
```

---

#### 6. Code — Code Block

Renders a fenced Markdown code block. Content is treated as raw text.

**Identifier:** `Code`

**Arguments:** `arg1` (optional) — Language identifier for syntax highlighting.

```js
@_Code_@: javascript;
const x = 42;
console.log(x);
@_end_@
```

**Output:**
````markdown
```javascript
const x = 42;
console.log(x);
```
````

---

#### 7. url — Hyperlink

Renders a Markdown inline link `[text](url "title")`.

**Identifier:** `url`

**Arguments:**
- `arg1` (required) — URL
- `arg2` (optional) — Link title

Body is the link display text.

```ini
[url = https\://example.com, Visit Example]Click here[end]
```

**Output:**
```markdown
[Click here](https://example.com "Visit Example")
```

---

#### 8. image — Image

Renders a Markdown inline image `![alt](url "title")`.

**Identifier:** `image`

**Arguments:**
- `arg1` (required) — Image URL
- `arg2` (optional) — Image title

Body is the alt text.

```ini
[image = photo.jpg, A nice photo]Landscape[end]
```

**Output:**
```markdown
![Landscape](photo.jpg "A nice photo")
```

---

#### 9. hr — Horizontal Rule

Renders a Markdown horizontal rule.

**Identifier:** `hr`

**Arguments:** `arg1` — Format character (`*`, `_`, or `-`). Defaults to `*`.

```ini
[hr][end]
[hr = _][end]
[hr = -][end]
```

**Output:**
```markdown
***
___
---
```

---

#### 10. escape — Escape Special Characters

Escapes all Markdown special characters in the content: `\ * _ { } [ ] ( ) # + - . ! > |`

**Identifier:** `escape`

```ini
[escape]Hello *World* _test_[end]
```

**Output:**
```markdown
Hello \*World\* \_test\_
```

---

#### 11. Table — Table

Renders a Markdown GFM table. Headers are defined as positional arguments; each body line is a row of comma-separated values. Content is treated as raw text.

**Identifier:** `Table`

**Arguments:** Column headers (positional)

```lua
@_Table_@: Name, Age, City;
Alice, 30, New York
Bob, 25, London
@_end_@
```

**Output:**
```markdown
| Name  | Age | City     |
|-------|-----|----------|
| Alice | 30  | New York |
| Bob   | 25  | London   |
```

---

#### 12. list — List

Passes content through as-is. Content is treated as raw text, so you write standard Markdown list syntax directly in the body.

**Identifier:** `list`

```lua
@_list_@;
- Apples
- Bananas
  - Cavendish
- Oranges
@_end_@
```

**Output:**
```markdown
- Apples
- Bananas
  - Cavendish
- Oranges
```

---

#### 13. todo — Todo Checkbox

Renders a Markdown GFM task list item.

**Identifier:** `todo`

**Block Usage:**
The body determines the state (`x` or `done` for checked).

```ini
[todo = Buy groceries]done[end]
```

**Output:**
```markdown
- [x] Buy groceries
```
