# Supported Output Formats

SomMark can convert `.smark` files into multiple formats. Each format serves a different purpose:

---

## 1. HTML

* **How to compile:**

```bash
sommark --html file.smark -o file.html
```

* **Programmatically:**

```javascript
const smark = new SomMark({ src, format: "html" });
const html = await smark.transpile();
```

---

## 2. Markdown

* **CLI:**

```bash
sommark --markdown file.smark -o file.md
```

* **Programmatically:**

```javascript
const md = await new SomMark({ src, format: "markdown" }).transpile();
```

---

## 3. MDX

* **CLI:**

```bash
sommark --mdx file.smark -o file.mdx
```

* **Programmatically:**

```javascript
const mdx = await new SomMark({ src, format: "mdx" }).transpile();
```

---

## 4. JSON

* **Use case:** For programmatic access, exporting document structure or metadata.
* **CLI:**

```bash
sommark --json file.smark -o file.json
```

* **Programmatically:**

```javascript
const json = await new SomMark({ src, format: "json" }).transpile();
```

---

## 5. Plain Text

* **Use case:** Simple text output, logging, or scripts that donâ€™t need formatting.
* **CLI:**

```bash
sommark --text file.smark -o file.txt
```

* **Programmatically:**

```javascript
const text = await new SomMark({ src, format: "text" }).transpile();
```

---

> [!Note]
> Only **one output format** can be used per command.
> You can **print to console** instead of saving using the `-p` flag:

```bash
sommark --html -p file.smark
```

* Using a **configuration file** (`smark.config.js`) allows custom mappers to affect output in all formats.
