# Installation

SomMark can be installed **globally** for CLI usage or **locally** for project-level API access.

---

## 1. Global Installation (CLI)

Use this if you want to run `sommark` commands directly from your terminal:

```bash
npm install -g sommark
```

**Verify installation:**

```bash
sommark --version
```

Once installed globally, you can run commands like:

```bash
sommark --html page.smark
```

---

## 2. Local Installation (For Projects / API)

Use this if you want to use SomMark in your code, for example to create custom mappers:

```bash
npm install sommark
```

### Simple Usage
The `transpile` helper is the easiest way to convert SomMark to HTML:

```javascript
import { transpile } from "sommark";

const html = await transpile({ src: "[Block]Hello[end]" });
console.log(html);
```

### Custom Mappers
For more control, you can create and register custom blocks:

```javascript
import { transpile, Mapper } from "sommark";

const myMapper = new Mapper();

myMapper.register("YouTube", function({ args }) {
  const id = this.safeArg(args, 0);
  return `<iframe src="https://youtube.com/embed/${id}"></iframe>`;
});

const output = await transpile({
  src: "[YouTube = dQw4w9WgXcQ][end]", 
  mapperFile: myMapper 
});
```

> [!TIP]
> Local installation is preferred for projects where you need custom mappers, helpers, or integration with build systems.
> Global installation is simpler if you just want to compile `.smark` files from the terminal.
