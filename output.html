CORE SPECIFICATION

The Language Standard.

Defining the formal grammar, structural constraints, and behavioral patterns of the SomMark syntax.

Primary Constructs

Blocks

The primary structural element. Supports hierarchical nesting and optional positional/key-value arguments.

Inline Statements

Used for phrase-level transformations like bold text. Cannot contains nested syntax.

AtBlocks

Specialized containers for raw, unparsed text. Perfect for source code snippets and literal data.

Comments

Single-line annotations starting with hash symbol. Interpreted or removed based on the target mapper.

Grammar Constraints

"At the top level, only Blocks and comments are permitted. All phrase content must reside within a valid Block scope."

Construct Matrix

Identifier Precision

Identifiers must be alphanumeric and may include dollar signs, underscores, or hyphens. Spaces are strictly prohibited.

PRACTICAL GUIDES

Mastering the Syntax.

Hands-on examples and best practices for building rich documentation with SomMark's core features.

1. Advanced Blocks

Blocks group content together and support flexible argument passing. Escape colons (`:`) if they appear inside a key-value value.

// Standard Block with Arguments 
    [Section = id: "about", class: "dark"]
        [h1]Welcome[end]
        This is structured content.
    [end]

2. Fluent Inline Statements

Transform short text fragments using identifiers. Inline arguments are comma-separated and do not support key-value pairs.

// Functional Inlines 
    [Block]
        [a = href: "https\://sommark.org", target: _blank]Click Here[end]
        (IMPORTANT)->(emphasis)
    [end]

3. Secure At-Blocks

Use At-Blocks for raw content that must bypass the parser. Remember to terminate the argument list with a semicolon (`;`).

// Raw Code Injection 
@_Code_@: javascript;
    function hello() {
        return "world";
    }
    @_end_@

4. Expert Escaping

"The backslash character '\' is your most powerful tool for rendering literal syntax characters."

To print a literal bracket, use '\['. To print a literal equal sign as the start of a block argument value, use '\='.
